## Done

- Scrape top 100 standards from JazzStandards.com → `raw.standards`.
- Query Spotify Web API for candidate recordings → `raw.recordings`.
- `raw` schema stores network data untouched; pipeline rebuilds without re-scraping.

## Decisions

- **DuckDB** — file-based, no server, no cost. `dbt-duckdb` adapter leaves the door open to BigQuery later without rewriting models.
- **No cleaning at ingestion** — typing, cleaning, and Spotify-result → standard matching are deferred to dbt, where they can be tested.
- **Scope** — first pass limited to the top 100.
